## Cambios por archivo

### `app/stores/auth.ts` — U1, U3, U4, U5, U15

El archivo de mayor impacto. Se aplicaron 5 mejoras simultáneas:

1. **`useLocalStorage` de VueUse** reemplaza la sincronización manual con `localStorage`. Los refs `session`, `token` y `tokenType` son ahora reactivos y SSR-safe por diseño, eliminando los guards `if (import.meta.client)` y la función `restoreSession()`.

2. **HMR support** agregado al final del archivo. Permite que los cambios en el store se apliquen en caliente sin recargar la página durante el desarrollo.

3. **console.error eliminados** (líneas 30 y 84). Los errores se propagan via objetos de respuesta estructurados.

4. **Bloque de logout comentado eliminado** (líneas 96–106). Código muerto que además contenía una URL hardcodeada inconsistente con U6.

5. **`restoreSession()` eliminada** del API público. Ya no es necesaria — `useLocalStorage` restaura automáticamente.

---

### `app/components/app-header.vue` — U2

```ts
// Antes
const { isAuthenticated, isLoading } = useAuthStore();

// Después
const authStore = useAuthStore();
const { isAuthenticated, isLoading } = storeToRefs(authStore);
```

**Crítico**: sin `storeToRefs`, los botones del header no reaccionaban a cambios de autenticación. Era un bug silencioso.

---

### `app/pages/login.vue` — U2, U5

```ts
// Antes
const { signIn, isAuthenticated } = authStore;  // bug: no reactivo
// ...
console.error('Login error:', ...);  // log de producción

// Después
const { signIn } = authStore;
const { isAuthenticated } = storeToRefs(authStore);
// ...

```

```ts

  public: {
    apiBase: 'http://erp.almacenadoravica.com/erp/public/api',
  },
```

Permite sobreescribir la URL con `NUXT_PUBLIC_API_BASE` en variables de entorno.

### `app/composables/use-api.ts` — U6

La constante hardcodeada `API_BASE_URL` fue reemplazada por `useRuntimeConfig().public.apiBase` en ambas funciones (`useApi` y `useApiFetch`).

### `app/pages/nova/catalogs/clients.vue` — U10

```ts
// Antes
```

### `app/middleware/auth.global.ts` — U16

El middleware fue simplificado: la llamada redundante a `restoreSession()` fue eliminada. La restauración de sesión es ahora responsabilidad exclusiva del store via `useLocalStorage`.
